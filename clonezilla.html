Clonación de Disco a Imagen

Clonezilla es una herramienta que nos permite copiar particiones o discos de memoria enteros a otros o a una imagen por ejemplo. Para ello necesitamos ejecutar el software desde un USB de arranque, proceso que ya se ha explicado en la web.

Desde la página web de Clonezilla podremos descargar el programa, pero tenemos que asegurarnos de elegir "iso" como "file type" para poder seguir el tutorial.

Como ya hemos comentado, tenemos que arrancar nuestro sistema desde un USB que contenga la iso de Clonezilla, para ello tenemos que acceder al gestor de arranque de nuestro equipo (normalmente pulsando F12 mientras se enciende el ordenador) y seleccionar dicho USB.

Al iniciar el sistema podemos seleccionar las opciones para ejecutar Clonezilla, elegiremos la primera.

Primero definimos el idioma con el que se mostrará la herramienta.

Lo siguiente a elegir es la distribución de teclado que queremos usar.

Prodemos hacer todo el proceso mediante linea de comandos si elegimos la opción "Enter_shell" pero elegiremos continuar con el asistente de Clonezilla.

Para el tipo de clonado seleccionaremos "device-image" para copiar un disco completo del equipo a una imagen iso.

Clonezilla puede copiar o recuperar una imagen iso en o desde multitud de ubicaciones, incluso en remoto como veremos más adelante, pero ahora seleccionaremos "local_dev" para guardar nuestra imagen en un disco conectado al sistema.

En esta vista de consola se nos muestran los dispositivos de memoria conectados al equipo, para identificar cual es el que queremos usar para almacenar nuestra imagen podemos conectarlo en este momento y aparecerá el último.

Ahora seleccionamos el disco donde se guardará nuestra imagen iso.

Si lo preferimos podemos chequear y reparar el estado del sistema de archivos, aunque solo funcionará si es de formatos usuales de Linux, no con NTFS.

Clonezilla necesita que le especifiquemos la ruta donde hacer la copia o de donde hacer la recuperación del disco.

Ejecutaremos el asistente con las opciones por defecto.

Decidimos si queremos hacer la imagen de un disco o de una partición.

Introducimos el nombre que le queremos dar a la imagen.

Selecionamos de qué disco haremos la imagen.

Tenemos un par de opciones de compresión pero seleccionaremos la opción por defecto.

De nuevo la opción de comprobar y reparar el sistema de archivos, pero en este caso de disco origen.

Para cuando complete la copia, podemos decidir si comprobar que se ha realizado correctamente.

Podemos cifrar la imagen por seguridad, pero en nuestro caso no es necesario.

Decidimos que hacer cuando el proceso se haya completado.

Al pulsar intro en la opción anterior comenzará el proceso de copia, se nos mostrará en esta consola las acciones que se vayan realizando, una vez completado todo el sistema hará lo que le hayamos indicado con anterioridad y ya tendremos nuestra copia realizada.

Recuperación de Disco desde Imagen

Para restaurar un disco desde una imagen usando Clonezilla, tendremos que seguir exactamente los mismos pasos hasta el momento en el que seleccionabamos el disco donde ubicariamos la copia, en este caso seleccionando el disco donde ya tenemos nuestra imagen creada.

Buscamos y seleccionamos la ubicación de las imagenes creadas con clonezilla a traves de este explorador de archivos.

Ahora seleccionamos "restoredisk" para restaurar nuestra imagen en un disco local.

Seleccionamos cual imagen querermos restaurar.

Elegimos el disco donde querermos hacer la restauración.

Podemos chequear si la imagen es válida para realizar la restauración.

Como al hacer la copia, al restaurar se nos mostrará por consola las acciones que el programa va realizando, una vez acabe, habremos completado la restauración.

Clonación de Disco a Imagen en Servidor Remoto

Si queremos realizar nuestra copia de disco almacenandola en un servidor remoto primero tendremos que crear una carpeta compartida en nuestro equipo destino. Para evitar errores es mejor dar permisos completos a todos los usuarios o al usuario en concreto que vaya ha realizar la copia. Para ello accedemos a las propiedades de la carpeta>compartir>uso compartido avanzado>permisos.

El proceso de hacer la copia en forma de imagen en un servidor remoto es igual que en local hasta el momento de elegir donde queremos crear la copia, elegiremos el protocolo samba en lugar de dispositivo local.

Elegiremos dhcp como forma de configurar la tarjeta de red del equipo, ya que es automática y no necesitamos otra configuración más avanzada.

Indicamos la dirección IP del equipo donde queramos almacernar nuestra copia.

Podemos definir un dominio, pero no tenemos así que seleccionamos "cancelar".

Elegimos la cuenta en el servidor que utilizaremos para guardar la imagen.

Designamos la carpeta comparida que creamos antes como ruta donde se escribiran los datos de la copia.

Podemos usar diferentes versiones del protocolo samba, usaremos la opción "auto".

También usaremos la opción "auto" al elegir el modo de seguridad.

Deberemos introducir la contraseña del usuario que designamos con anterioridad para hacer la copia en el servidor.

El resto de pasos son iguales a la copia en local, una vez completemos la configuración, se realizará la copia a través de la red.

Si nos sale esta información o similar en la consola al finalizar la configuración, se debe a que no hemos establecido correctamente los permisos de la carpeta compartida como se indica en el primer paso de esta guía.

Recuperación de Disco desde Imagen en Servidor Remoto

Para la restauración desde un servidor remoto el proceso es el mismo que la restauración en local pero seleccionando el protocolo samba en lugar de un disco local y configurando la IP y demás referente al servidor como ya hemos visto en la creación de imagen en remoto.
